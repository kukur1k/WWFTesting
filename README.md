# DevProjectManager
## Описание
Консольное приложение DevProjectManager предназначено для работы с рабочими проектами и ведения их списка. Основная функция - чтение данных о проектах и запись их в файл из различных форматов (JSON, YAML, XML, CSV). Присутствует  функционал для добавления, изменения, сортировки и поиска проектов (с фильтрацией по свойствам). 
## Класс `DevProject` - основная модель данных проекта
### Конструктор `DevProject(string name, string leader, Status status, int priority)` 
- инициализирует объект DevProject (Сам проект)
### `ResetID() `
- сбрасывает счетчик событий
### `override string ToString()`
- переопределенный метод для вывода проекта
### ` SetCurrentID(int devProjectID)`
- метод для присвоения значения индексу (используется при пересчете нового индекса после чтения из файла)
### Класс `Projects` - хранит список проектов (объекты типа DevProject)
### Класс `WorkwithData<T>` - универсальный класс с методами для сериализации/десериализации
###  `WriteToFile<T>(List<T> obj, string filePath)`
- Универсальный метод для записи в файл
- Параметры:
  - `obj`: список объектов типа Т для записи
  - `filePath`: путь к файлу
### `ReadData(string filepath)`
- Универсальный метод для чтения из файла
- Параметры:
  - `filePath`: путь к файлу

## Класс `ServicesMethods` - статический класс с методами для манипуляции данными
### `Sort(List<DevProject> Projects, string ParameterName)`
- Метод для сортировки данных (возвращает отсортированный массив объектов)
- Параметры:
  - `filePath`: путь к файлу
  - `obj`: список объектов типа Т для записи
  - `Projects`: список объектов типа DevProjects
  - `ParameterName`: наименование параметра для сортировки
### `Search(string subString, List<DevProject> devProjects)`
- Метод для поиска данных по подстроке (по полям Name и Leader)
- Параметры:
  - `devProjects`: список объектов типа DevProjects
  - `subString`: подстрока для поиска
### `AddDevProject(Projects projects, DevProject proj)`
- Метод для добавления проекта (объекта DevProject) в список проектов
- Параметры:
  - `projects`: объект класса Projects (список проектов в который необходимо добавить проект)
  - `proj`: проект для добавления
### `UpdateDevProject(int devProjectID, Projects projects, string newName, string newLead, Status newStatus, int newPriority)`
- Метод для обновления данных проекта
- Параметры:
  - `devProjectID`: ID изменяемого проекта
  - `projects`: объект класса Projects (список проектов в который необходимо добавить проект)
  - `newName` - новое наименование проекта
  - `newLead` - новый лидер
  - `newStatus` - новый статус
  - `newPriority` - новое приоритет проекта
### `RemoveDevProject(int devProjectID, Projects projects)`
- Метод для удаления проекта из списка проектов
- Параметры:
  - `projects`: объект класса Projects (список проектов из которого необходимо удалить проект)
  - `devProjectID`: ID удаляемого проекта

## Тестирование
Консольное приложение включает в себя набор автоматизированных тестов, реализованных с использованием фреймворка NUnit. Тесты находятся в классах `TestProlectManager_ReadAndWrite` и `TestProjectManager_CRUDOperations`
### В классе `TestProlectManager_ReadAndWrite` тестируются основные операции сериализации/десериализации, записи и чтения
### Тестируются следующие аспекты:
- Запись и чтение для формата JSON 
- Запись и чтение для формата XML
- Запись и чтение для формата YAML
- Запись и чтение для формата CSV
### В классе `TestProjectManager_CRUDOperations` тестируются основные операции c данными
### Тестируются следующие аспекты:
- Добавление проекта
- Удаление проекта
- Поиск по подстроке
- Сортировка по параметрам:
  - По Наименованию
  - По Лидеру
  - По Приоритету
- Обновление данных проекта

## В приложении реализовано консольное меню со следующими опциями:
- Считывание данных из файла;
- Запись данных в файл;
- Вывод данных на экран;
- Сортировка данных по выбранному параметру;
- Поиск данных по подстроке;
- Добавление данных в структуру;
- Удаление данных из структуры;
- Изменение данных структуры;
